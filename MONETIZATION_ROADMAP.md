# DocListen 웹앱 수익화 로드맵

## 목표
현재 DocListen MVP를 기반으로, 돈을 받을 수 있는 베타 서비스 → 정식 수익화 서비스로 단계적으로 전환한다.

## 현재 상태
- 웹앱 MVP 있음
- PDF 업로드 가능
- PDF 페이지 보기 가능
- 문단 클릭 재생 가능
- 브라우저 기본 TTS로 낭독 가능
- 같은 PDF 마지막 위치 이어보기 가능
- 현재 링크는 LocalTunnel 임시 링크라 정식 서비스용은 아님

## 수익화 기본 방향
광고만으로 시작하지 않는다. PDF 듣기는 사용량과 가치가 명확하므로 아래 구조가 더 적합하다.

1. 무료 체험
2. 유료 베타
3. 구독 또는 크레딧
4. 이후 광고는 보조 수익으로 검토

## 추천 가격 가설

### 무료
- 하루 PDF 1개
- 짧은 PDF만 가능
- 브라우저 기본 음성
- 이어읽기 가능

### 베타 유료
- 월 4,900원 또는 9,900원
- 긴 PDF 가능
- 더 많은 PDF 사용
- 저장/이어듣기
- 광고 없음
- 향후 고급 음성/요약 기능 제공 예정

### 크레딧형 후보
- 10개 PDF 변환권: 4,900원
- 30개 PDF 변환권: 9,900원
- 긴 PDF는 추가 차감

## 단계별 진행

### 1단계: 수익화 검증용 랜딩 페이지 만들기
목표: 사용자가 돈 낼 가치가 있는지 확인한다.

필요 작업:
- 홈 화면에 서비스 소개 추가
- 무료/유료 플랜 섹션 추가
- 베타 신청 버튼 추가
- 문의/신청 연결 추가
- 개인정보/파일정책/이용약관 링크 정리

완료 기준:
- 사용자가 링크만 보고 무엇을 파는 서비스인지 이해 가능
- 가격이 보임
- 베타 신청 또는 문의 행동을 할 수 있음

### 2단계: 정식 도메인 준비
목표: LocalTunnel 임시 링크에서 벗어난다.

필요 작업:
- 도메인 후보 정하기
- 도메인 구매
- Render/Railway/Fly.io/VPS 중 배포 위치 선택
- HTTPS 적용

완료 기준:
- `https://서비스도메인`으로 접속 가능
- 휴대폰에서 안정적으로 열림

### 3단계: 파일 정책/약관/개인정보 페이지 정리
목표: 사용자가 PDF 업로드에 대해 불안하지 않게 만든다.

필요 페이지:
- 개인정보처리방침
- 이용약관
- 파일 처리 정책
- 문의 페이지

핵심 문구:
- 업로드한 PDF를 어디에 저장하는지
- 음성 변환용으로만 쓰는지
- 자동 삭제 시점
- 사용자가 삭제 요청할 방법

완료 기준:
- 서비스 하단에서 모든 정책 페이지 접근 가능
- PDF 업로드 전 파일 처리 원칙이 보임

### 4단계: 유료 베타 신청 받기
목표: 자동 결제 전, 수동으로 돈 낼 사람이 있는지 검증한다.

방법:
- 카카오톡 오픈채팅 또는 이메일 문의
- 구글폼 신청
- 계좌이체/수동 결제
- 베타 이용자 5~10명 확보

완료 기준:
- 최소 1명 이상 결제 의사 확인
- 어떤 기능 때문에 돈을 내는지 파악

### 5단계: 로그인/사용량 제한
목표: 무료/유료 구분이 가능하게 만든다.

필요 기능:
- 이메일 로그인 또는 간편 로그인
- 하루 사용량 카운트
- 무료 사용자 제한
- 유료 사용자 제한 해제

완료 기준:
- 무료 사용자는 제한에 걸림
- 유료 사용자는 더 많이 사용 가능

### 6단계: 자동 결제 연결
목표: 반복 결제가 가능하게 만든다.

후보:
- 토스페이먼츠
- PortOne/아임포트
- Stripe, 해외 결제용

완료 기준:
- 결제 성공 시 유료 권한 부여
- 결제 실패/취소 처리 가능

### 7단계: 광고/애드센스 검토
목표: 무료 사용자에게 보조 수익을 만든다.

전제 조건:
- 정식 도메인
- 정보성 콘텐츠
- 개인정보/약관/문의 페이지
- 충분한 페이지 품질

완료 기준:
- 애드센스 심사 신청 가능

## 가장 먼저 할 일
1. 웹앱 첫 화면에 유료 베타 안내 추가
2. 가격표 추가
3. 베타 신청/문의 버튼 추가
4. 파일 처리 안내 문구 추가
5. 링크로 확인

## 이번 주 MVP 목표
- `수익화 검증용 랜딩 페이지` 추가
- 무료/유료 플랜 보여주기
- 베타 신청 CTA 추가
- 정식 배포 전 체크리스트 완성

## 보류할 것
아직 바로 하지 않는다.
- 복잡한 회원가입
- 자동 결제
- 앱스토어/플레이스토어 출시
- 애드센스 신청
- 고급 AI 음성 대규모 적용

먼저 돈 낼 사람이 있는지 검증한다.
