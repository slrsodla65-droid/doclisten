# PDF 듣기 서비스 이름/도메인 후보

## 추천 1순위: DocListen

이유:
- PDF뿐 아니라 문서 전체로 확장 가능
- 영어권/한국 사용자 모두 이해 쉬움
- `listen`이 서비스 핵심 기능을 바로 설명함
- 나중에 PDF, 문서, 웹페이지, 리포트 읽기 서비스로 확장 가능

도메인 후보:
- doclisten.app
- doclisten.io
- doclisten.co.kr

1차 DNS 확인:
- 현재 DNS 연결 없음
- 단, 구매 가능 여부는 도메인 판매처에서 최종 확인 필요

## 추천 2순위: PDF Listen

이유:
- 가장 직관적
- 사용자가 뭘 하는 서비스인지 즉시 이해 가능
- 검색 키워드에도 유리함

단점:
- 너무 일반적인 이름이라 브랜드화는 약할 수 있음
- PDF 외 문서로 확장할 때 이름이 좁아질 수 있음

도메인 후보:
- pdflisten.app
- pdflisten.io
- pdflisten.co.kr

## 추천 3순위: ReadMyPDF

이유:
- 사용자가 원하는 행동과 정확히 맞음
- “내 PDF를 읽어줘”라는 느낌이 좋음
- 랜딩페이지 문구 만들기 쉬움

단점:
- 이름이 조금 김
- 한국 사용자에게는 PDF Listen보다 직관성이 약간 떨어질 수 있음

도메인 후보:
- readmypdf.app
- readmypdf.io
- readmypdf.co.kr

## 추천 4순위: VoicePDF

이유:
- PDF를 음성으로 바꾼다는 의미가 명확함
- 짧고 기억하기 쉬움

단점:
- 약간 기능명처럼 들림
- 고급 서비스 느낌은 DocListen보다 약함

도메인 후보:
- voicepdf.app
- voicepdf.io
- voicepdf.co.kr

## 추천 결론

가장 추천하는 이름은 `DocListen`.

이유:
1. PDF에 갇히지 않음
2. 문서 듣기 서비스라는 방향이 명확함
3. 나중에 유료 서비스/앱/브랜드로 키우기 좋음
4. `DocListen - 문서를 귀로 듣다` 같은 카피가 가능함

## 다음 실행 순서

1. 사용자가 이름 1개 선택
2. 도메인 판매처에서 구매 가능 여부 확인
3. 가능하면 `.app` 먼저 구매
4. 한국 타깃이면 `.co.kr`도 후보
5. 정식 배포는 `Render` 또는 `Railway` 우선 검토

## 구매 우선순위

1. doclisten.app
2. doclisten.co.kr
3. pdflisten.app
4. readmypdf.app
5. voicepdf.app
